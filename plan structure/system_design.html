👤 M4 (Bridge)
Pillar 1 · Core
🔬
Stylometric Analyzer
AI Engine · M1 (Captain)
Deep linguistic fingerprinting of every paragraph. Detects writing style shifts that indicate multiple authors.
  • Per-paragraph profiling: vocabulary richness, avg sentence length, passive voice %, formality score, technical density, hedging frequency
  • Pairwise comparison: consecutive paragraph similarity scoring (0–100)
  • Author clustering: group paragraphs by style into K clusters
  • Author count estimation: minimum # of distinct authors
  • Boundary detection: sharp style drops = authorship change points
GPT-4o-mini NLTK stylometry
⏱ ~15s per paragraph
Pillar 2 · Core
📚
Citation Forensics Engine
Backend · M2 (Engine)
Analyzes reference patterns for temporal and thematic inconsistencies — a strong signal of post-hoc assembly.
  • Reference extraction: parse all citations from paper text and bibliography
  • Temporal analysis: do sections cite different eras? (2019 vs 2024 papers mixed)
  • Thematic clustering: do citation topics shift abruptly between sections?
  • Cross-section consistency: score citation coherence across the paper
  • Anomaly flagging: sections with isolated, unrelated reference clusters
GPT-4o regex parsing clustering
⏱ ~5s per analysis
Pillar 3 · Completeness
🔍
Semantic Source Tracer
Integration · M4 (Bridge)
Cross-references flagged sections against academic databases to find probable source papers — providing traceable evidence.
  • Embed flagged paragraphs: convert suspicious text to vector embeddings
  • arxiv search: query arxiv API with extracted key phrases
  • Semantic matching: cosine similarity between embeds and arxiv abstracts
  • Source ranking: top-K probable source papers with similarity score
  • Evidence linking: map flagged section → source paper with % match
text-embedding-3-small arxiv API cosine sim
⏱ ~10s per flagged section
